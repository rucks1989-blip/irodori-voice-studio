話者切替

文章での使用例

【佐藤】こんにちは。【花子】おはよう。

【佐藤】 → 佐藤.wav と登録すると、【佐藤】を検出した位置から佐藤の音声になります。検出文字は自由記入で、＜佐藤＞ や @@佐藤 でも構いません。読みやすいため 【佐藤】 形式を推奨します。

複数人のWAVをまとめて登録

佐藤.wav、花子.wav、太郎.wavのように、何人分でも追加できます。

登録済みWAV0件
登録済みWAVはありません。上のボタンから複数選択できます。
検出文字とWAVの明示的な対応

開始記号・終了記号の決まりはありません。原稿中で検出したい文字をそのまま入力します。

明示的な対応はありません。
SPLIT

自然な分割

最大文字数を超えた場合、。 → ！？ → 、 → 強制の順で自然な位置を後方探索します。安全側の初期値は120文字です。

FILTER

読まない文章

登録した文字列だけを原稿から取り除き、その後の本文は読み上げます。

SYSTEM

audio.cpp接続

このUIはaudio.cppを交換可能なバックエンドとして利用します。設定はこのPCだけのsettings.local.jsonへ保存されます。

CHAT

キャラクター会話

LLMを追加すると会話ができます

OpenAI互換APIを起動して接続先を設定してください。現在の実機例は、接続名 character-chat（GGUFメタデータ: Gemma 4 E2B 4.6B）です。

初回の返答生成は、モデルの読み込みにより時間がかかる場合があります。 生成後にLLMを終了する設定では、次回の会話でも再読み込み時間が発生します。

「リリー」は動作確認用のサンプルです。
キャラクターを選び、メッセージを送ってください。
OPTIONAL

LLM起動例

llama.cppでOpenAI互換APIを起動する例です。モデルは同梱されません。

  --host 127.0.0.1 --port 11438 \
  -c 4096 -ngl 99
