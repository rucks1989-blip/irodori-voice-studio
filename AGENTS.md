Repository Editing Rules

このリポジトリを編集するAI・自動化ツールは、以下を必ず守ること。
文字コードを壊さない

編集前に対象ファイルの以下を確認すること。

    文字コード

    BOMの有無

    改行コード

    日本語の有無

既存ファイルの文字コード、BOM、改行コードを、必要なく変更してはならない。
Windows PowerShell 5.1

.ps1へ日本語を追加する場合、Windows PowerShell 5.1で正常に読み込める形式を必ず確認すること。

UTF-8 BOMなしの.ps1は、内容がASCIIのみなら動作していても、日本語追加後にWindows PowerShell 5.1でパース不能になる場合がある。

日本語を含む.ps1は、原則としてUTF-8 BOM付きで保存し、以下を実行して確認すること。

$errors = $null
[void][System.Management.Automation.Language.Parser]::ParseFile(
    (Resolve-Path ".\対象ファイル.ps1"),
    [ref]$null,
    [ref]$errors
)
$errors

パーサーエラーが1件でもある状態で作業を完了してはならない。
Windowsバッチ

.batと.cmdは、既存の文字コードとCRLFを維持すること。

日本語を含む場合、実際のcmd.exeで表示と実行を確認すること。

理由なくLFへ変換してはならない。
安全な編集

以下を禁止する。

    文字コード未確認でのファイル全体書き換え

    BOMの無断追加または削除

    改行コードの一括変換

    Get-Content | Set-Contentによる無確認の上書き

    Pythonのopen()でencoding未指定の書き込み

    日本語を含むファイルへのASCII前提の処理

    文字化けした内容のまま保存

    ???、 、縺などの異常を無視すること

編集後の確認

日本語を含む編集対象について、作業終了前に必ず確認すること。

    UTF-8として正常に読み込める

    U+FFFDが含まれない

    既存日本語行が不必要に消えていない

    BOMが意図どおり

    改行コードが意図どおり

    git diffで無関係な全文変更がない

    .ps1はWindows PowerShell 5.1でパースできる

    .batはcmd.exeで実行できる

事故時の報告

今回の編集によって文字コード、BOM、改行コードを壊した場合、既存不具合として報告してはならない。

以下を明記すること。

    自分のどの編集で発生させたか

    修正前後の文字コード

    BOMの変化

    改行コードの変化

    日本語本文が破壊されたか

    どの方法で修復したか

    他の編集対象も監査したか

作業範囲

対象ディレクトリー外を変更してはならない。

一時ファイルも、可能な限りリポジトリ内の専用一時ディレクトリーへ作成すること。

OSのTEMPを使用した場合は、自分が作成したものだけを識別し、作業終了時に原状回復すること。
