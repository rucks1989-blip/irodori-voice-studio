# irodori voice studio

> **開発中のローカル音声生成ツールです。**
> このREADMEは公開準備用のたたき台です。特にライセンス、配布物、対応環境については、正式公開前に各配布元の最新情報を再確認してください。

audio.cpp版Irodori-TTSを、ブラウザーから手軽に利用するためのWindows向けローカルUIです。

単に文章を読み上げるだけでなく、長文の自然な分割、話者ごとの参照WAV切替、読まない文字列の除去、チャンク間の無音挿入、生成WAVの結合までをまとめて行います。通常の音声生成にLLMは必要ありません。

## できること

- 入力文章を読みやすい位置で分割して音声生成
- 1チャンク最大120文字を目安にした後方探索分割
- `。`、`！`、`？`、`、`などを考慮した分割
- 分割位置に応じた短い無音の挿入
- 任意の文字列を読み上げ対象から除外
- 原稿中の検出文字による話者切替
- 複数の参照WAV登録と明示的な対応付け
- その生成だけで使う参照WAVの選択
- 生成前の分割プレビュー
- 生成音声の再生とWAV保存
- キャラクターJSONを使った会話機能（LLMは別途必要）
- ローカルAPIからの音声生成

話者切替の検出文字は固定形式ではありません。例えば `【佐藤】`、`＜佐藤＞`、`@@佐藤` など、原稿に合わせて自由に登録できます。読みやすさの面から `【名前】` 形式を推奨しています。

## 動作方針

このソフトはローカル利用を前提としています。

- UI、音声生成API、audio.cppとの通信は初期設定ではすべて `127.0.0.1` を使用します。
- 登録したWAV、入力した文章、生成音声、キャラクターJSONを外部サービスへ送信する機能は設けていません。
- オプションのLLM接続先を外部APIへ変更した場合、その会話内容の扱いは接続先サービスの規約に従います。

## 必要なもの

- Windows 10またはWindows 11
- Python 3
- audio.cpp音声生成バックエンド
- audio.cppで利用できるIrodori-TTS GGUFモデル
- 必要に応じて、使用する権利を持つ参照WAV

会話機能を使う場合のみ、OpenAI互換Chat APIを提供するLLMバックエンドと、そのバックエンドで使用できるモデルが別途必要です。LLMモデルはこのリポジトリに同梱しません。

## 初回セットアップ

1. このリポジトリをダウンロードまたは展開します。
2. `setup.bat` をダブルクリックします。
3. 表示されたライセンス、ダウンロード内容、PC環境への変更内容を確認します。
4. 同意できる場合のみセットアップを続行します。
5. セットアップ結果に未達成項目が表示された場合は、同時に作成される案内を確認します。
6. 準備が完了したら `start.bat` をダブルクリックします。

初回セットアップ後の起動は `start.bat`、終了は `stop.bat` を使用します。未セットアップの状態で `start.bat` を押した場合は、先にセットアップが案内されます。

セットアップが自動導入できない項目は `download_links.html`、詳しい補足は `SETUP_MANUAL.txt` に記載しています。

### セットアップが行う可能性のある処理

利用者の確認後、環境に応じて次の処理を行う可能性があります。

- Python 3.12の導入案内またはwingetによるインストール
- audio.cpp公式Windowsパッケージのダウンロード
- Irodori-TTS GGUFモデルのダウンロード
- 必要なPythonパッケージのインストール
- ローカル設定ファイルと実行用設定の作成

ダウンロード対象には可能な範囲で固定リビジョンとSHA-256検証を使用しています。ただし、セットアップを実行する前にスクリプトと配布元を確認してください。

## GPUについて

GPU対応範囲はaudio.cppの配布ビルド、CUDAランタイム、NVIDIAドライバーによって変わります。

現在のセットアップ処理は、対応条件を満たす環境では公式CUDA版、それ以外では公式CPU版を選択します。GTX 10シリーズなどPascal世代向けのCUDAビルドは自動導入対象ではありません。Pascal対応版を自分で用意した場合は、外部audio.cppバックエンドとして接続できます。

GPU対応条件は将来変更される可能性があります。正式公開前および導入時に、必ずaudio.cppの最新Releaseと説明を確認してください。

## 基本的な使い方

### 通常の音声生成

1. 「音声生成」画面へ文章を入力します。
2. 必要なら「今回使う参照WAV」からWAVを選択します。
3. 「分割を確認」で読み上げ単位を確認します。
4. 「音声を生成」を押します。
5. 生成後、画面上で再生するかWAVを保存します。

参照WAVを選択しない場合は、Irodori-TTS内部の音声が使用される場合があります。初回生成はモデルや参照音声の読み込みにより時間がかかることがあります。

### 話者切替

1. 「話者切替」で使用するWAVを登録します。
2. 検出文字とWAVの対応を追加します。
3. 原稿中に同じ検出文字を記述します。

例：

```text
【佐藤】こんにちは。
【花子】おはようございます。
```

`【佐藤】` を `佐藤.wav`、`【花子】` を `花子.wav` に対応付けると、検出位置から次の話者まで参照音声が切り替わります。検出文字自体は読み上げません。

### 読まない文字列

設定した文字列を音声生成前に原稿から取り除けます。作品独自の注釈や、読み上げたくないラベルなどに利用できます。

## オプションのキャラクター会話

「会話」タブはOpenAI互換Chat APIへ接続し、LLMの返答を通常の音声生成処理へ渡します。返答にも分割、参照WAV、無音挿入、WAV結合が適用されます。

動作確認用として `characters/lily.sample.json` を同梱しています。これはキャラクターJSONの記述例であり、音声やLLMモデルは含まれていません。自作JSONは画面から追加できます。

初期設定例：

- Chat API: `http://127.0.0.1:11438/v1/chat/completions`
- Health API: `http://127.0.0.1:11438/health`
- Model alias: `character-chat`

使用するLLMおよびGGUFのライセンス、安全性、必要スペックは利用者自身で確認してください。

## ローカルAPI

初期ポート：

- UI: `http://127.0.0.1:6670`
- API: `http://127.0.0.1:6666`
- audio.cpp: `http://127.0.0.1:8091`

主なエンドポイント：

- `GET /api/health` — 接続状態の確認
- `POST /api/plan` — 分割結果の確認
- `POST /api/generate` — 音声生成

最小の生成例：

```powershell
$body = @{ text = "こんにちは。" } | ConvertTo-Json
Invoke-RestMethod `
  -Uri "http://127.0.0.1:6666/api/generate" `
  -Method Post `
  -ContentType "application/json" `
  -Body $body
```

現時点のAPIはローカル利用向けです。認証、アクセス制限、レート制限など、外部公開サーバー向けの保護機能は備えていません。LANやインターネットへ直接公開しないでください。

## ローカルに保存されるもの

次のデータは実行時に作成されますが、Gitの公開対象から除外されています。

- `settings.local.json` — 利用者固有の設定
- `data/` — 登録WAV、履歴、セットアップ状態
- `outputs/` — 生成音声
- `models/` — ダウンロードしたモデル
- `runtime/` — audio.cpp実行環境と一時設定
- `logs/` — ログとプロセス情報

公開前には次の検査を実行できます。

```powershell
python scripts/public_safety_check.py
```
