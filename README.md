# irodori voice studio

**Windows向けローカル音声生成ツール（開発中）**

Irodori-TTS（audio.cpp版）をブラウザーから手軽に利用するためのローカルUIです。

GTX 1070（VRAM 8GB）環境での動作を確認しており、長文の自動分割、話者切替、参照WAV管理、無音挿入、WAV結合までをまとめて行えます。
VRAM足りない、そもそもどうやって使うかわからないという人向けです。
通常の音声生成にはLLMは不要です。初心者向け視点でcodexとchatgpt使って開発しています。自分が初心者なので初心者がどうやれば使えるか。
ライセンス的なのをAIに調べさせ自動でダウンロードできるもの、できないものを分けています。

## 主な特徴

- Windows向けローカルUI
- LLMなしで音声生成可能
- ChatGPT・Gemini・Claudeなどで作成した文章をそのまま音声化
- 話者タグによる参照WAV自動切替
- Irodori-TTS向け長文自動分割
- 分割プレビュー
- WAV結合・無音挿入
- REST API対応
- GTX 1070（VRAM 8GB）で動作確認済み

## LLMは必須ではありません

このソフト単体で音声生成できます。

例えばChatGPTやGeminiで作成した文章をそのまま貼り付けるだけで利用できます。

```text
【佐藤】こんにちは。

【花子】おはようございます。

【佐藤】今日はいい天気ですね。
```
話者タグに対応する参照WAVを登録しておけば、自動で話者を切り替えて読み上げます。
ストーリーテラーやキャラ同士の会話などchatgptやgeminiのカスタムで作ってそれをそのまま貼り付ければそれだけで楽しく使えると思います。
具体的なやり方はAIに聞いた方が早いので各AIにやり方を聞いてください。あくまでここはその文章をコピペするのがメインです。
キャラ会話に関しては別途記載しています。

## 長文自動分割

Irodori-TTSは長時間の連続生成では品質が低下する場合があります。

本ソフトでは、次の基準で自動分割し、長文でも自然につながるよう設計しています。

- 約120文字を目安
- 句読点を優先
- 不自然になりにくい位置

## セットアップ

### 必要なもの

- Windows 10 / 11
- Python 3
- audio.cpp
- Irodori-TTS GGUFモデル
- 参照WAV（任意）

初回のみ `setup.bat` を実行してください。

セットアップ完了後は `start.bat` で起動できます。

自動導入できなかった項目は `download_links.html`、詳しい補足は `SETUP_MANUAL.txt` を参照してください。

## 初心者向けサポート

`setup.bat` の実行後には、セットアップ結果画面が表示されます。

画面では現在のPC環境を自動で確認し、次の情報などを一覧で確認できます。

- OS
- GPU
- NVIDIAドライバーバージョン
- Compute Capability
- 現在使用している音声バックエンド（CUDA / CPU）
- 導入済みの項目
- 未達成・推奨項目
- 任意で追加できる機能

また、**「未達成項目をAI相談用にコピー」**ボタンを用意しています。

このボタンを押すと、現在のPC環境と不足している項目をまとめた質問文がクリップボードへコピーされます。

そのままChatGPT、Gemini、ClaudeなどのAIへ貼り付けるだけで、次の内容を現在の環境に合わせて案内してもらえます。

## どうしても解決しない場合（最終手段）

通常は setup.bat が現在のPC環境や不足している項目を確認し、必要なダウンロード先や設定方法を案内します。また、「未達成項目をAI相談用にコピー」ボタンから、現在の環境に合わせた質問文を ChatGPT、Gemini、Claude などのAIへそのまま貼り付けて相談できます。

それでも解決しない場合は、最終奥義 AIに丸投げ.bat を実行してください。

この機能は、通常のセットアップや診断でも解決できなかった場合の最終手段として用意しています。

実行すると、次のファイルを自動で作成します。

    診断ZIP

    AIへ貼り付ける文章.txt

作成された AIへ貼り付ける文章.txt と 診断ZIP を、そのまま ChatGPT、Gemini、Claude などのAIへ添付してください。

診断ZIPには、AIが問題を調査しやすいよう、問題解決に必要な最小限の情報が含まれます。ログからディレクトリーも読み取るので各々の環境に適したものになっています。

AIはこれらを確認し、

    導入済みで再利用できるもの

    足りないもの

    壊れている設定

    パスの食い違い

    自動取得できるもの

    手動で導入が必要なもの

    修正した方がよいコードや設定

などを整理し、現在の環境に合わせた修正方法を提案できます。

    注意

    AIは実際にあなたのPC上でプログラムを実行して調査するわけではありません。そのため、ログに現れない故障や、外部ソフト・ドライバー・バイナリの内部状態などは確定できない場合があります。

    その場合は、AIから提案された修正版を適用して再実行し、新しい診断ZIPを作成して再度AIへ相談することで、段階的に原因を絞り込むことを想定しています。

    AIの機能や回答内容は日々変化します。分からないことや判断に迷うことがあれば、そのままAIへ相談してください。

## 基本的な使い方

1. 原稿を入力
2. 必要なら参照WAVを選択
3. 「分割を確認」でチャンクを確認
4. 「音声を生成」
5. 再生または保存

参照WAVを選択しない場合は、Irodori-TTS内部の音声が使用される場合があります。

初回生成はモデルや参照音声の読み込みにより時間がかかることがあります。
geminiのgemでストーリーテラーとかを作ると個人的には面白いと感じてます。

## 話者切替

話者切替画面で、次のように対応付けを登録します。

```text
【佐藤】 → satou.wav
【花子】 → hanako.wav
```

原稿中の話者タグは読み上げず、登録した参照WAVへ自動的に切り替わります。

## 読まない文字列

指定した文字列を音声生成前に削除できます。

作品独自の注釈やラベルなど、読み上げ不要な文字列に利用できます。

## オプション：キャラクター会話

通常の音声生成にLLMは不要ですが、「会話」タブではOpenAI互換Chat APIを利用できます。

例：

```text
http://127.0.0.1:11438/v1/chat/completions
```

LLMの返答も通常の音声生成処理へ渡されるため、次の機能をそのまま利用できます。

- 自動分割
- 話者切替
- 無音挿入
- WAV結合

## GPUについて

GPU対応はaudio.cppのビルドに依存します。

現在は、次の両方に対応しています。

- CPU版
- CUDA版

GTX 1070（Pascal・VRAM 8GB）環境で動作を確認しています。

ローカルLLMを利用する場合は、次の運用を想定しています。

```text
LLM起動
↓
文章生成
↓
LLM終了（VRAM解放）
↓
Irodori-TTSで音声生成
```

この方式により、VRAMが少ない環境でもLLMと音声生成を切り替えながら利用できます。
